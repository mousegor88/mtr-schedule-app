# React Native 新手讀碼順序（給 C# Developer）

目標：30~60 分鐘內掌握這個專案的「入口、導航、頁面、共用元件、設定」。

---

## 0) 先建立心智模型（2 分鐘）

- 這是 Expo + React Native + expo-router 專案
- expo-router 是「檔名即路由」
- app/ 底下每個 .tsx 幾乎都可視為一個 page/screen

對照 C#：
- app/*.tsx ≈ MVC 的 Action + View（合在一起）
- components/*.tsx ≈ 可重用 UI 元件
- hooks/*.ts ≈ 可重用邏輯 helper
- app.json / eas.json ≈ 應用與發佈設定

---

## 1) 看 package.json（3 分鐘）

檔案：package.json

先看三件事：
1. "main": "expo-router/entry"（路由入口）
2. scripts（start / android / ios / web / lint）
3. dependencies（expo-router、react-native、expo-*）

你會知道這是什麼技術棧、怎麼跑起來。

## 2) 看根路由：app/_layout.tsx（5 分鐘）

這檔案等於全站導航殼（shell）。

## 4) 看首頁流程：app/(tabs)/index.tsx（8 分鐘）

## 5) 看一個完整功能頁：先讀 app/tko.tsx（15 分鐘）

檔案：app/tko.tsx

照這個順序讀：
1. 型別定義：RawTrain / DisplayTrain
2. API_URL 與 DEST_MAP
3. mapDestination / mapNextTrain / mapTrains（資料轉換）
4. state（loading/error/data/secondsLeft）
5. load()（fetch + error handling）
6. useEffect（首次載入 + setInterval 自動更新）
7. renderSection + JSX 表格
8. StyleSheet（只看命名，不要逐行糾結）

掌握後再看：
- app/lhp.tsx（單方向版本）
- app/tbc.tsx（與 tko 類似）

---

## 新手常見卡點（你可以特別注意）

1. 把 route 當成傳統集中設定：
   - 在 expo-router 是「檔名即路由」，不是手寫 routes map。

2. 把資料與 UI 分太開：
   - RN 常在同一個 screen 內管理資料抓取 + 顯示（小專案很常見）。

3. 過度關注 style 細節：
   - 第一輪先看資料流（API -> map -> state -> render），再看 UI 微調。

---

## 建議實作練習（30 分鐘）

1. 在首頁新增一個按鈕導到 /modal
2. 在 tko.tsx 把 REFRESH_SECONDS 從 15 改成 10（理解自動更新機制）
3. 在 mapDestination 加一個站碼映射
4. 在 app.json 改版本號並重新啟動

做完你就會對這個專案的主要骨架很熟。



產生android
首次運行
npm install -g eas-cli
確保 expo.json or app.json 的 Android 配置部分完整
eas login
eas init

再次運行
build android 
eas build -p android --profile preview
or
eas build -p android --profile release

preview 產出 APK（方便測試與 side-load）。
release 預設產出 AAB（適合 Play 上架）。
觸發建置
產出 APK（測試安裝用）：eas build -p android --profile preview
產出 AAB（上架用）：eas build -p android --profile release
第一次建置會詢問簽章金鑰（keystore），可選擇「讓 EAS 代管」。
下載成品
建置完成後，EAS 會提供下載連結。把 APK 裝到裝置即可測試。
注意：Google Play 對「新 App」要求上傳 AAB，不接受 APK 作為上架包。
本機打包（不走雲端）

適用：你想離線打包，或有自訂原生模組。
作法：
先將 Expo 專案「預建」原生專案：npx expo prebuild
用 Android Studio 開啟 android 資料夾，或在命令列執行：
Windows/macOS/Linux：cd android && ./gradlew assembleRelease
產物位置：android/app/build/outputs/apk/release/app-release.apk（或 outputs/bundle/release/app-release.aab）
需求：安裝 JDK、Android SDK、Android Studio（或具備相同 CLI 工具）。
